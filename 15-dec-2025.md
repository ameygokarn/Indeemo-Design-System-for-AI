## Key Design Decisions & Rationale

### 1. **Color Space Consistency**
- All non-neutral colors use **HSL color space** for modifiers
- Only **cream** uses SRGB (intentional design choice)
- HSL provides consistent perceptual lightness progression

### 2. **Modifier Progression Logic**
- **First step (100)**: Maximum lightness (99%) for near-white tints
- **Light progression**: Gradual decrease in lightening amount
- **Base color (750)**: No modifier - represents the pure brand color
- **Dark progression**: Gradual increase in darkening amount  
- **Last step (1050)**: Near-black shade (1.8% lightness)

### 3. **Token Studio First Approach**
- All changes made with Token Studio export compatibility in mind
- `$themes` and `$metadata` sections preserved and validated
- Token set order maintained for proper Figma variable creation

### 4. **Accessibility Considerations**
- Lightness extremes (99% and 1.8%) provide maximum contrast range
- Each step calculated for perceptual uniformity
- WCAG contrast data maintained in descriptions
